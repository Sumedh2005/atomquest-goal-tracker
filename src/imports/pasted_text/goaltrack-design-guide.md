 GoalTrack Portal
Design a clean, professional web application called GoalTrack — an internal employee goal setting and performance tracking portal. The visual style should feel like a modern SaaS dashboard built by a thoughtful human designer: flat surfaces, generous whitespace, a single indigo-purple accent on a neutral base. Think Linear, Vercel dashboard, and Notion — not Salesforce or any legacy HR tool. Every screen must feel calm, focused, and trustworthy.

DESIGN PHILOSOPHY & AESTHETIC DIRECTION
This portal must feel like a product designed by a thoughtful human designer, not generated by AI. The goal is calm, focused, and trustworthy — the kind of internal tool employees actually enjoy using, not one they dread opening.
Minimal by default. Every element on screen must earn its place. If it doesn't help the user complete a task, remove it. No decorative illustrations, no gradient hero sections, no icon clusters, no colored sidebar backgrounds. Whitespace is a design element — use it deliberately.
One color does the heavy lifting. The entire UI is built on a neutral base (white cards, #F7F7F6 page background, #E8E8E6 borders) with a single accent — #4F46E5 indigo-purple. That accent appears only on: the active nav item, primary action buttons, progress fills, and key data highlights. Everything else is grayscale. This restraint is what makes it look professional, not AI-generated.
Typography carries the hierarchy. Use size and weight contrast instead of color to create visual hierarchy. Page titles at 20px/500, section labels at 13px/500 uppercase tracked, body and table text at 14px/400, captions at 12px/400. Never use more than two font sizes on a single card.
Tables over cards where data is dense. The instinct of AI tools is to put everything in cards. Resist this. Goal lists, team lists, audit logs, reports — these should be clean tables with subtle row separators, not stacked cards. Cards are reserved for single bounded objects like one employee's goal sheet or one metric summary.
Status badges are the only color. The only place colors other than the primary accent appear is status badges — green for approved/completed, amber for pending/in-progress, red for overdue/error, gray for draft/not started. These badges should be small, pill-shaped, with a light tint background and dark same-family text color. Never use full-saturation background colors.
Interaction feedback is subtle. Hover states use a #F3F3F2 background shift, not color changes. Focus rings use the primary accent at low opacity. Active nav items get a left 3px border accent in primary purple, not a filled background pill.
Forms must feel effortless. Input fields have a light #F0F0EE background with no border by default, switching to a subtle 1px border plus white background on focus. Labels sit above inputs in 12px/500. Helper text and validation messages appear inline below in 12px, never as modal popups. No jarring alert dialogs.
The weightage bar is the hero moment. On the goal creation screen, the live segmented weightage bar filling to 100% is the one place where design can delight. Each goal segment gets a progressively lighter shade of the primary accent — not random colors. A clean percentage counter sits beside it. This should feel satisfying and responsive — the single standout interaction in an otherwise utilitarian tool.
References: Linear (table density, typography), Notion (whitespace, calm), Vercel dashboard (minimal nav, metric cards), Liveblocks (restrained color). Explicitly avoid: Salesforce, SAP, Workday, any legacy enterprise HR aesthetic.

GLOBAL ELEMENTS
Top navigation bar: logo + wordmark "GoalTrack" on the left, active cycle name in the center (e.g. "FY 2025–26 · Q1 Check-in Open"), user avatar + name on the right with a small role badge below the name (Employee / Manager / Admin).
Left sidebar: 200px wide, white background, no colored fill. Nav items are 14px with a Phosphor or Lucide outline icon left of the label. Active item has a 3px left border in #4F46E5 and a #F3F3F2 row background. Section groupings in 11px uppercase tracked gray labels. No nested dropdowns.
Main content area: #F7F7F6 background. Content max-width 1100px centered. Cards are white, 1px solid #E8E8E6 border, 12px border radius, 20px 24px padding. Page title in 20px/500, page subtitle in 14px/400 muted gray directly below.

SCREEN 1 — Login Page
Centered card 420px wide on a #F7F7F6 background. GoalTrack logo + name at top centered. Tagline in 13px muted: "Your goals, tracked and aligned." Email input, password input, primary purple "Sign in" button full width. A thin divider with "or" below, then a "Sign in with Microsoft" button with the Microsoft four-square logo, outlined style. No role selector — role is server-determined. Minimal footer: "GoalTrack · Powered by your organization."

SCREEN 2 — Employee: Create Goal Sheet
Page title "Create goal sheet" · subtitle "FY 2025–26 · Goal setting window closes 31 May."
A repeating goal entry card for each goal added (max 8). Each card has a subtle #1 goal number label top-left in muted gray. Fields inside in a 2-column grid:

Row 1: Thrust Area (dropdown), Unit of Measurement (dropdown with 4 options: Numeric – Higher is better, Numeric – Lower is better, Timeline, Zero-based)
Row 2: Goal Title (full width text input)
Row 3: Description (full width textarea, 3 lines)
Row 4: Target Value (text input), Weightage % (number input, min 10)
Trash icon top-right of each card to remove

Below all goal cards: a full-width "Weightage summary" section — not a card, just an inline block. A segmented horizontal bar 8px tall, divided by goal, each segment a shade of indigo. Below the bar: individual goal labels with their percentage. Total shown right-aligned: "Total: 87%" in amber if not 100%, "Total: 100%" in green when complete.
"+ Add another goal" text button left-aligned, disabled and shows "Maximum 8 goals reached" tooltip when limit hit.
Sticky bottom action bar (not floating, just a border-top section): "Save as draft" outlined button left, "Submit for approval" filled indigo button right — disabled with tooltip "Weightage must equal 100%" until valid.

SCREEN 3 — Employee: My Goals (Locked View)
A subtle green info banner at the top of the page: "Your goals have been approved and are now locked. Contact your admin for any changes." with a lock icon.
Below: a single clean table (not cards) with columns: #, Goal Title, Thrust Area, UoM, Target, Weightage. Weightage column shows both the percentage and a small inline bar. Footer row of the table shows "Total weightage: 100%" right-aligned in green. No edit controls visible.

SCREEN 4 — Employee: Quarterly Check-in
Page title "Q1 check-in" · subtitle "July window · Enter your actual achievement for each goal."
A window status pill at the top right: "Window open · Closes 31 Jul" in green.
A table with columns: Goal Title, UoM, Planned Target, Actual Achievement (editable input), Status (dropdown: Not Started / On Track / Completed), Progress Score (auto-computed read-only pill). Progress score color logic: ≥90% green, 60–89% amber, <60% red, not yet entered gray dash.
Below table: "Save check-in" filled button right-aligned. A note below the button: "You can update your actuals until the window closes."

SCREEN 5 — Employee: Dashboard
4 metric summary tiles in a row. Each tile: muted label 12px above, large number 28px/500 below. Tiles: Total Goals, Weightage (green if 100%), Approved Goals, Q1 Updated (e.g. 3 of 6). No card borders on these tiles — just a #F0F0EE background fill.
If check-in window is open: a slim indigo banner below the tiles: "Q1 check-in window is open — update your achievements before 31 July." with a right-aligned "Go to check-in →" link.
Below: goal progress table. Columns: Goal, Thrust Area, Target, Q1 Actual, Score, Status. Compact rows, 14px text, alternating #FAFAFA row background on hover only.

SCREEN 6 — Manager: Approvals Queue
Page title "Pending approvals" · subtitle "3 goal sheets awaiting your review."
A table listing employees who submitted: Avatar initials circle, Name, Department, Goals submitted, Submitted date, a "Review" button per row (outlined, small). Clicking "Review" expands an inline panel below that row (accordion style, not a modal) showing the full goal table for that employee. Inside the expanded panel: each goal row has inline-editable Target and Weightage fields (click to edit, shows a pencil icon on hover). A weightage total row at the bottom validates live. Two action buttons at the bottom of the panel: "Return for rework" (amber outlined) and "Approve & lock" (green filled). A comment text area appears conditionally only when "Return for rework" is clicked.

SCREEN 7 — Manager: Team Dashboard
4 metric tiles: Team size, Pending approvals (amber number if >0), Q1 check-ins done (e.g. 5/8), Goals locked.
Below: a team member table. Columns: Avatar + Name, Department, Goals, Approval status badge, Check-in status badge, Q1 avg score bar (a thin inline progress bar + percentage). Clicking any row navigates to that employee's full goal detail view.

SCREEN 8 — Manager: Check-in Review
A list of team members. Each row shows: name, avatar, Q1 update status (submitted / not submitted), and a "Review" button. Clicking opens a split layout — left two-thirds is a read-only table of planned vs actual per goal with computed scores, right one-third is a comment panel with a textarea labeled "Check-in comment" and a "Submit comment" button. After submission the row is marked "Check-in done" with a green badge.

SCREEN 9 — Admin: Organization Dashboard
6 metric tiles in 2 rows of 3: Total Employees, Sheets Submitted, Pending Approvals, Managers with Pending, Q1 Check-ins Done %, Overall Completion Rate.
Below: a horizontal bar chart (simple, clean, no chart borders) showing check-in completion by department. Each bar is indigo with a percentage label at the end.
Below the chart: a full employee table with columns: Name, Department, Manager, Approval Status, Check-in Status, "Unlock Goals" action button (appears only if goals are locked — clicking triggers a confirmation inline).

SCREEN 10 — Admin: Cycle Configuration
Page title "Cycle configuration" · subtitle "FY 2025–26."
A table: Phase, Window Opens, Deadline, Status badge, Edit button. Status badges: gray Closed, green Active, blue Upcoming. An "Add phase" button top-right. Editing a row expands it inline with date pickers and a save button — no modals.

SCREEN 11 — Admin: Shared Goals
A two-section layout. Top section: a form to create a shared goal — Goal Title, Target, UoM, then a multi-select searchable dropdown for department or individual employees. A "Push to employees" button. A note below the form: "Recipients can adjust weightage only. Title and target are read-only for them."
Bottom section: a table of existing shared goals — Title, Target, Pushed to (department badge + count), Primary owner, Linked employees count.

SCREEN 12 — Admin: Achievement Report
A filter bar: Department, Quarter, Status dropdowns + search input. Right side of the bar: "Export CSV" and "Export Excel" outlined buttons with icons.
A sortable table: Employee Name, Department, Manager, Goal Count, Q1 Avg Score (colored badge), Check-in Status. Clicking a row expands it showing individual goal scores inline.

SCREEN 13 — Admin: Audit Trail
A filter bar: date range picker + employee name search.
A chronological list — not cards, just clean rows with a thin separator. Each entry: a small diff or edit icon left in muted gray, action title in 14px/500, who performed it in 13px muted (Role · Name), what changed as "old value → new value" in monospace 13px, timestamp right-aligned in 12px muted. No borders on entries — just 16px vertical padding and a 1px separator.

DESIGN SYSTEM
Colors:

Page background: #F7F7F6
Card background: #FFFFFF
Border: 1px solid #E8E8E6
Primary accent: #4F46E5 (indigo-purple)
Primary light tint: #EEF2FF
Success: text #166534, background #F0FDF4
Warning: text #92400E, background #FFFBEB
Danger: text #991B1B, background #FEF2F2
Muted text: #6B7280
Secondary text: #9CA3AF
Primary text: #111827

Typography:

Font: Inter
Page title: 20px / 500
Card title: 15px / 500
Section label: 11px / 500 / uppercase / tracked
Body + table: 14px / 400
Caption + helper: 12px / 400
Weights used: 400 and 500 only — never 600 or 700

Spacing: 4px base unit. Use 8, 12, 16, 20, 24, 32px. Generous vertical padding between sections (32px minimum).
Components:

Buttons: 36px height, 14px, 8px border radius, primary filled uses #4F46E5, outlined uses transparent bg with #E8E8E6 border
Inputs: #F0F0EE bg, no border at rest, 1px solid #4F46E5 on focus, 8px radius, 36px height
Badges: 5px 10px padding, 20px border radius pill, 12px/500 text
Tables: no outer border, 1px row separators only, 40px row height, hover #FAFAFA
Metric tiles: #F0F0EE background, no border, 8px radius, 16px padding
Avatars: 32px circle, #EEF2FF background, #4F46E5 initials text, 13px/500

Icons: Lucide or Phosphor outline style only. 16px inline, 20px standalone. Never filled icons. Muted gray color except when used as an action (then primary accent).
